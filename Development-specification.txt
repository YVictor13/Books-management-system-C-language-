课题名称	         图书管理系统

Ⅰ、设计目的和要求：
1.1设计目的
随着人们生活水平的日益提高，人们对文化的需求也越来越大。在人们的业余生活中，越来越多的人乐于享受文字给生活带来的乐趣。而图书也是学生的学习过程中必不可少的。而本项目旨在研发一个图书馆的管理系统，实现图书馆的信息化管理。在方便读者查询借阅图书的同时，方便图书馆管理员对图书进行管理。
1.2 设计题目要求
1.2.1 设计要求
本系统提供登陆系统模块，对学生老师以及管理员不同登录状态做了不同的处理，使登陆界面更加人性化，也更方便管理员进行管理。本系统提供添加书目，删除书目，编辑书目信息等操作，方便后台管理员对所有图书进行管理。针对电脑管理图书的需求设计的，可以完成读者登记、增删书籍、查看图书的类型，图书类型说明、修改图书馆信息和查看本图书馆的介绍等主要功能
1.2.2 功能要求
1、添加图书类别
2、更改图书类别
3、添加图书
4、更改图书
5、退出系统
6、图书馆介绍与联系方式
1.3 系统的具体需求
1.3.1 登录主界面
用类DButil来实现主界面的显示，用DbUtil类中的BUtil(String username,String pasword)方法链接数据库核对用户名和密码是否正确，如果正确返回1，如果输入不存在返回2，如果读取数据库失败返回3。
1.3.2 用户注册界面
用类dao来实现登录界面的显示，用DBUtil类中的public Connection getCon() throws Exception方法链接数据库，将新用户添加入数据库。
1.3.3 用户主页界面
用类public class LogOnFrm来实现主页界面的显示，主页中包括图书类别添加、图书添加，关于我们，图书类别修改、图书修改和最后有安全退出按钮
图书类别添加与图书类别修改：用类DBUtil来实现取款窗口的显示，用Denlu类中的public Connection getCon() throws Exception方法链接数据库操作数据。
图书添加与图书修改：用类BookTypeManageInterFrm来实现存款窗口的显示，用Denlu类中的public BookManageInterFrm()方法链接数据库操作数据。
安全退出：用类public MainFrm() 来实现转账窗口的显示，用dao类中的public Connection getCon() throws Exception方法链接数据库操作数据。

			private void loginActionPerformed(ActionEvent evt){
		String userName=userNameTxt.getText();
		String password=new String(passwordTxt.getPassword());
		if(StringUtil.isEmpty(userName)){
			JOptionPane.showMessageDialog(null, "用户名不能为空");
			return;
		}
		if(StringUtil.isEmpty(password)){
			JOptionPane.showMessageDialog(null, "密码不能为空");
			return;
		}
		User user=new User(userName,password);
		Connection con=null;
		try {
			con=dbUtil.getCon();
			User currentUser = userDao.login(con, user);
			if (currentUser != null) {
				dispose();
				new MainFrm().setVisible(true);
			} else {
				JOptionPane.showMessageDialog(null, "用户名或密码错误");
			}
		} catch (Exception e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
			JOptionPane.showMessageDialog(null, "登录失败");
		}finally{
			try {
				dbUtil.closeCon(con);
			} catch (Exception e) {
				e.printStackTrace();
			}
		}
	}

Ⅶ、总结与心得
	此次通过使用JAVA来进行程序课程设计图书管理系统给了我很大的启示，以前没有用JAVA自己编辑过系统，第一次遍还是很茫然的。最后到网上查看了别人编写的代码，脑海中才有了系统的雏形。然后自己设计流程图将复杂的系统分为几个模块，一个一个解决感觉还是挺轻松的。在此过程中图形界面比较好做，但是访问数据库时经常出现自己想不到的问题，语法、语义错误还是经常有的，最后在网络上的教程的帮助下解决了所有问题。在刚刚接触到这次项目的时候，我对于这个项目并没有多少头绪，初期工作的进展也异常缓慢，但是随着对实训项目的逐步深入，我总结出实训项目的大体结构框架，包括对实训项目中所遇问题的总结，以及查询资料，请教老师，团队合作的这些举措下，解决了各种各样的问题。像是类的使用，信息的增、删、改、查，主显示页面的友好性以及数据持久化等等实现都渐入佳境。长达三周的Java语言的学习也为我成功实现该实训项目打下坚实的基础，并且我应该感谢网上教程的老师，老师为我答疑解惑，而与同学，我们共同探讨，相互支持，他们对于我这次使用Java项目的实践都是不可或缺的，无论失去哪一方，相信我都无法取得好的结果。这次使用Java来做项目的实践对于我来说真的是一次难得的经历与体验。希望之后的学习生涯中还能有同样的机会去接触这样的程序设计的项目，提升自己，度过一段充实的时光。
本次程序设计课程主要侧重在对数据库的添加、删除、修改、查看等最基本的操作，由于本学期刚好学习数据库，通过此次项目，知道了掌握这些最基本的设计思想是很重要的，能够帮助我们开发更加复杂的软件提供依据和基础，所以看似简单的课程设计，却是对我们设计软件的一次考验，一次磨练，所以我一直都保持着很认真地精神来进行设计的，在今后的学习里，一定要更加努力，学好这门编程语言。
